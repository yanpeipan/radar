# Milestone v1.1: GitHub Monitoring

**Shipped:** 2026-03-23
**Status:** Complete

## Overview

GitHub Monitoring milestone adds the ability to track GitHub repository releases and changelog files, displaying them alongside RSS feed articles in a unified interface.

## Requirements

- [x] **GH-01**: User can add a GitHub repository URL to monitor
- [x] **GH-02**: System fetches release information using GitHub API (tag_name, body, published_at, html_url)
- [x] **GH-03**: System supports GitHub token authentication via environment variable (GITHUB_TOKEN)
- [x] **GH-04**: System handles GitHub API rate limits gracefully (60 req/hour unauthenticated, 5000 req/hour with token)
- [x] **GH-05**: System detects changelog files (CHANGELOG.md, HISTORY.md, CHANGES.md) via raw.githubusercontent.com
- [x] **GH-06**: System scrapes changelog content and stores as article
- [x] **GH-07**: New releases and changelog changes are displayed in unified format
- [x] **GH-08**: System reuses existing refresh mechanism (fetch --all includes GitHub sources)

## Key Decisions

1. **GitHub Releases via REST API** — Using GitHub REST API `/repos/{owner}/{repo}/releases/latest`
2. **Bearer token auth** — GITHUB_TOKEN env var enables 5000 req/hour
3. **Changelog via raw.githubusercontent.com** — HEAD detection + GET content fetching
4. **UNION ALL for unified listing** — Combining feed articles and GitHub releases in list_articles()
5. **LIKE search for GitHub releases** — github_releases.body not in FTS5, using LIKE instead
6. **FTS sync pattern** — INSERT INTO articles_fts using SELECT from articles table
