## v1.8 Completed Requirements

**Defined:** 2026-04-04
**Milestone:** v1.8 OpenClaw 本地测试与 Skill 迭代

### 基础流程 (FUNDAMENTAL)

- [ ] **FUND-01**: `openclaw run feedship-ai-daily` 能被 OpenClaw 正确加载和执行
- [ ] **FUND-02**: `feedship fetch --all` 正常抓取所有订阅源
- [ ] **FUND-03**: `feedship article list --since YYYY-MM-DD --limit 100` 正确按日期过滤
- [ ] **FUND-04**: `feedship search` 语义搜索能返回相关结果

### Cron 与 Isolated Session (CRON)

- [ ] **CRON-01**: `openclaw cron add` 能创建 feedship-ai-daily cron job
- [ ] **CRON-02**: cron job 在 isolated session 下 `feedship` 命令 PATH 可达
- [ ] **CRON-03**: `--announce` flag 正确把报告输出投递到配置好的 channel
- [ ] **CRON-04**: `openclaw cron run <job-id>` 能立即触发 cron job

### 报告格式 (REPORT)

- [ ] **REPORT-01**: 报告包含完整的 A~F 六个 section
- [ ] **REPORT-02**: Section E (创业信号) 能从现有文章中提取到有效的创业相关内容
- [ ] **REPORT-03**: Section F (创作点) 能从现有文章中提取到有效的内容角度
- [ ] **REPORT-04**: 空数据场景 ("今日无新文") 能正确处理并给出友好提示

- [ ] **CHANNEL-02**: 频道投递的格式（markdown）能在飞书端正确渲染

- [ ] **EDGE-01**: feedship 未安装时，skill 能给出清晰的安装指引
- [ ] **EDGE-02**: ml/cloudflare extras 缺失时能检测并提示安装
- [ ] **EDGE-03**: OpenClaw gateway 未启动时能检测并提示启动命令
- [ ] **EDGE-04**: 超时场景（大量文章或慢速网络）能有合理的处理方式

## Out of Scope

| Feature | Reason |
|---------|--------|
| `--id` 和 `--url` 同时使用 | 场景不明确，用户意图混淆 |
| 数据库存储 `--url` 抓取结果 | 一次性抓取不应污染数据库 |
| 支持所有 provider 开放抓取 | 仅支持已有 provider 的 URL |
