**Key Decisions:**
- Use Nitter RSS over Twitter API (no auth required)
- NitterProvider priority 300 matches GitHubReleaseProvider tier
- Auto-discover Nitter instance to avoid single-point-of-failure
- JSON output replaces Rich entirely when --json passed
- Consistent object wrapper format across all commands

**Issues Resolved:**
- Nitter fetch timeout issues (reduced to 15s with network_idle=False)
- Support for twitter: and x: pseudo-URLs
- ChromaDB metadata handling for datetime objects and floats
