# Requirements: Feedship v1.7 — OpenClaw AI Daily Report

**Defined:** 2026-04-04
**Core Value:** 用户能够在一个地方集中管理所有资讯来源，无需逐一访问各个网站。

## v1 Requirements

### 定时推送 (DAILY)

- [ ] **DAILY-01**: 用户可以通过 `openclaw cron add` 配置每日定时任务，触发 feedship ai-daily 报告生成和推送
- [ ] **DAILY-02**: 定时任务使用 `--session isolated` 模式运行，避免上下文污染
- [ ] **DAILY-03**: 定时任务使用 `--announce` 标志确保报告结果被投递到指定渠道
- [x] **DAILY-04**: ai-daily SKILL.md 中包含完整的 cron 配置示例和说明

### 日报模板 (REPORT)

- [x] **REPORT-01**: 日报支持按来源（feed）分组展示文章
- [x] **REPORT-02**: 日报包含 AI 生成的摘要（3-5 条要点）
- [x] **REPORT-03**: 日报支持按阅读量/相关度排名筛选文章
- [x] **REPORT-04**: 日报模板可通过 ai-daily skill 文档查看和配置

### AI 挖掘 (MINE)

- [x] **MINE-01**: 日报包含「创业信号」板块 — 挖掘融资、收购、趋势性机会
- [x] **MINE-02**: 日报包含「创作点」板块 — 挖掘可写的主题、角度、热点
- [x] **MINE-03**: AI 挖掘结果以结构化格式呈现，便于推送

### 多渠道发送 (CHANNEL)

- [x] **CHANNEL-01**: 支持 Telegram 渠道推送（via openclaw channel）
- [x] **CHANNEL-02**: 支持飞书渠道推送（via openclaw channel）
- [x] **CHANNEL-03**: ai-daily SKILL.md 包含渠道配置指南（Telegram + 飞书）
- [x] **CHANNEL-04**: 渠道选择通过 OpenClaw 灵活配置，无需修改 feedship 代码
