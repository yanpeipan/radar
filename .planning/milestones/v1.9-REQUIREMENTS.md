# Requirements: Feedship

## v1.9 Active Requirements

**Defined:** 2026-04-05
**Milestone:** v1.9 fetch --url

### Fetch Command Enhancement

- [ ] **FETCH-01**: `feedship fetch --url <URL>` 可以直接抓取 URL，不保存到数据库
- [ ] **FETCH-02**: `--json` 输出包含 `articles` 字段，返回所有抓取的 article 详情
- [ ] **FETCH-03**: 支持 GitHub Trending URL（如 `https://github.com/trending?since=weekly`）
- [ ] **FETCH-04**: `--url` 和 `--id` 互斥，不能同时使用
- [ ] **FETCH-05**: URL 不需要先 `feed add`，直接可抓取
- [ ] **FETCH-06**: 错误处理完善（无效 URL、无 provider、fetch 失败）

### Search Enhancement (v2)

- **SRCH-01**: `feedship search --url <URL>` 直接搜索 URL 内容（不依赖数据库）
- **SRCH-02**: 支持 Tavily API 搜索

## v1.10 Completed Requirements

**Milestone:** v1.10 article view 增强 (SHIPPED: 2026-04-06)

### Article View Enhancement

- [x] **VIEW-01**: `feedship article view --url <URL>` — 直接抓取 URL，Trafilatura 提取内容，返回 Markdown，不入库
- [x] **VIEW-02**: `feedship article view --id <article_id>` — 从数据库查 article，抓取 link，Trafilatura 回填 content 字段，更新数据库，返回内容
- [x] **VIEW-03**: `feedship article view --json` — JSON 格式输出（--url/--id 共用）
- [x] **VIEW-04**: Trafilatura 最佳实践：output_format=markdown，include_images=False，include_tables=True

## Out of Scope

| Feature | Reason |
|---------|--------|
| `--id` 和 `--url` 同时使用 | 场景不明确，用户意图混淆 |
| 数据库存储 `--url` 抓取结果 | 一次性抓取不应污染数据库 |
| 支持所有 provider 开放抓取 | 仅支持已有 provider 的 URL |
