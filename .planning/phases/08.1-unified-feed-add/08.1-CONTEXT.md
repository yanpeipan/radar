# Phase 8.1: Unified Feed Add - Context

**Gathered:** 2026-03-23
**Status:** Ready for planning

<domain>
## Phase Boundary

让 `feed add` 命令支持 GitHub blob URL，自动检测并委派给 changelog 流程。添加成功后，GitHub URL 作为 feed 存入数据库，changelog 内容作为 article 存储。`fetch --all` 可刷新所有 feed（包括 GitHub URL feed）。

</domain>

<decisions>
## Implementation Decisions

### 架构
- **D-01:** 修改 `src/feeds.py` 中的 `add_feed()` 函数，在开头检测 URL 类型
- **D-02:** GitHub blob URL → 走 changelog 分支；RSS/Atom URL → 走原有 RSS 解析分支
- **D-03:** 使用 `src/crawl.py` 中已有的 `is_github_blob_url()` 做 URL 类型检测

### 存储模型
- **D-04:** GitHub URL feed 的 feed_id 使用随机 UUID（与现有 RSS feed 相同）
- **D-05:** GitHub URL 本身作为 feed.url 存储
- **D-06:** changelog 内容通过 `detect_changelog_file()` + `fetch_changelog_content()` 获取后，存入 articles 表

### 错误处理
- **D-07:** 未检测到 changelog 文件时，返回明确错误信息，不添加 feed
- **D-08:** 网络错误（如超时、403 等）→ 透传原始错误信息

### 刷新逻辑
- **D-09:** `fetch --all` 时，对 GitHub URL feed 调用 `refresh_changelog()`（复用 Phase 5 的逻辑）

</decisions>

<specifics>
## Specific Ideas

用户场景：
```
python -m src.cli feed add https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md
```
该命令应该成功添加 feed 并抓取 changelog 作为 article。

验证条件：
```
python -m src.cli feed list  # 能看到新增的 GitHub URL feed
python -m src.cli fetch --all  # 不报错，刷新 changelog
```
