# Implementation decisions from 16-CONTEXT.md

## GitHubReleaseProvider (D-01, D-06, D-08, D-09, D-10, D-11)
- File: src/providers/github_release_provider.py
- Class: GitHubReleaseProvider
- Self-registers via PROVIDERS.append() at module import time
- match() — same URL matching as GitHubProvider (github.com HTTPS and SSH)
- priority() — returns 200 (higher than GitHubProvider's 100)
- crawl() — calls repo.get_latest_release() via PyGithub, returns [{tag_name, name, body, html_url, published_at}]
- parse() — same Article structure as GitHubProvider.parse()
- Empty list on any error (rate limit, not found, etc.)
- Uses _get_github_client() pattern from github_provider.py
- Uses parse_github_url() from github_utils.py

## ReleaseTagParser (D-02, D-12, D-13)
- File: src/tags/release_tag_parser.py
- Class: ReleaseTagParser with parse_tags(article) method
- Self-registers via TAG_PARSER_INSTANCE at module import time
- tag_parsers() on provider returns [ReleaseTagParser()]
- Extracts tags:
  - owner tag: from URL via parse_github_url()
  - release tag: always added for release articles
  - version tags: extracted from tag_name using regex (e.g., v1.2.3 -> v1, v1.2, v1.2.3)
  - release type tags: major-release if tag matches x.0.0, minor-release if x.y.0, bugfix-release otherwise
</context>
