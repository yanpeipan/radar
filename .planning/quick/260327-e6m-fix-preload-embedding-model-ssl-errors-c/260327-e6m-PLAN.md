<objective>
Fix preload_embedding_model SSL errors crashing all CLI commands. article view and other non-semantic commands should work without network/model access.
</objective>

<context>
Bug: preload_embedding_model() in src/cli/__init__.py crashes when SSL/network errors occur during model download. This happens even for commands like `article view` that don't need semantic search.

Root cause: src/cli/__init__.py line 32-33 calls preload_embedding_model() unconditionally for every CLI command.

Fix: Wrap preload_embedding_model() in try/except in both the CLI caller and the function itself. If preload fails, log a warning and continue — the model will be downloaded on first semantic search use.
</context>
