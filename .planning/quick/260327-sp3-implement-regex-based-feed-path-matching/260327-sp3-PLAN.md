## Implementation

Add `generate_feed_candidates(base_url: str) -> list[str]` to common_paths.py that:
1. Extracts base URL (scheme + host + port)
2. Generates root-level candidates from regex patterns (pattern → direct path)
3. Generates subdirectory candidates by substituting _FEED_SUBDIR_NAMES into wildcard patterns

Root patterns that generate candidates directly:
- `^/feed/?$` → "/feed"
- `^/rss$` → "/rss"
- `^/rss\.xml$` → "/rss.xml"
- `^/atom\.xml$` → "/atom.xml"
- `^/feed\.xml$` → "/feed.xml"
- `^/index\.xml$` → "/index.xml"

Subdirectory patterns (substitute subdir names):
- `^/[^/]+/rss\.xml$` → "/{subdir}/rss.xml" for subdir in ("feed", "rss", "blog", "news", "atom", "feeds")
- `^/[^/]+/atom\.xml$` → "/{subdir}/atom.xml" for subdir in ("feed", "rss", "blog", "news", "atom", "feeds")
- `^/[^/]+/feed\.xml$` → "/{subdir}/feed.xml" for subdir in ("feed", "rss", "blog", "news", "atom", "feeds")

## Important Notes

The regex patterns in `_FEED_PATH_PATTERNS` are for MATCHING discovered URLs (in `_extract_links`). The `generate_feed_candidates()` function is for GENERATING candidates to probe. They serve different purposes but share the same underlying path concepts.
