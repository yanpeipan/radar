## Decisions

1. **Context managers over inline Progress** — `FetchProgress` and `DiscoverProgress` encapsulate all progress bar state, making `_fetch_with_progress` much simpler (9 lines → 5 lines).
2. **elapsed_time returned from FetchProgress** — The context manager tracks start time internally, exposes `elapsed_time` as a property computed on demand.
3. **DiscoverProgress wraps full discovery** — Since `discover_feeds` returns a list (not a generator), the progress updates happen after the fact as feeds are enumerated.
4. **print_summary kept separate** — Not folded into `FetchProgress` because fetch callers need to call it after the context exits.
