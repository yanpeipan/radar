# 260328-59g: Revert CLI simplification from 260328-522

## Summary

Reverted `list_articles()` and `search_articles()` to return `list[ArticleListItem]` instead of `list[dict]`. The `print_articles()` function was updated to handle both `ArticleListItem` (for list/search) and `dict` (for semantic search via ChromaDB).

## Changes

### src/application/articles.py

- **ArticleListItem dataclass**: Added `score: float = 1.0` field as last field with default value
- **list_articles()**: Now returns `list[ArticleListItem]` directly from `storage_list_articles()`
- **search_articles()**: Now returns `list[ArticleListItem]` directly from `storage_search_articles()`
- Removed import of `rank_list_results`, `rank_fts_results`, `format_articles` from `src.application.search`

### src/application/search.py

- **print_articles()**: Updated to accept `list[Any]`, using `hasattr(item, '__dataclass_fields__')` to distinguish:
  - `ArticleListItem`: uses `item.score`, `item.feed_name`, `item.pub_date`, `item.link`, `item.description`
  - `dict` (semantic): uses `item.get("score", 1.0)`, `item.get("url")` for source/domain, `item.get("pub_date")`, `item.get("document")`
