## What was built

Added `cluster_articles_for_report_v2` and `render_report_v2` functions that produce a topic-clustered report structure supporting three sections: layers (A), signals (B), and creation (C).

## Key Changes

### `src/application/report.py`

- `_classify_signal_leverage()` / `_classify_signal_business()` / `_classify_creation()` — rule-based classification helpers
- `cluster_articles_for_report_v2()` — returns `{"layers": [...], "signals": {leverage, business}, "creation": [...], "date_range": {...}}`
- `render_report_v2()` — renders v2.md template with new data structure
- `get_topic_title_chain()` added to chains.py

### `src/cli/report.py`
- `--template v2` routes to `cluster_articles_for_report_v2` + `render_report_v2`

### `~/.config/feedship/templates/v2.md`
- New template with section A (五层蛋糕), B (创业信号), C (创作选题)

## v2 Data Structure

```python
{
  "layers": [{"name": str, "topics": [{"title": str, "sources": [{"title": str, "url": str}]}]}],
  "signals": {"leverage": [...], "business": [...]},
  "creation": [{"name": str, "topics": [...]}],
  "date_range": {"since": str, "until": str}
}
```
