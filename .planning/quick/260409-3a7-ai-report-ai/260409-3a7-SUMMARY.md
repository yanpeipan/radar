## What Was Built

### Fix #3: SQLite writes removed from asyncio.gather
Both v1 (`_cluster_articles_async`) and v2 (`_cluster_articles_v2_async`) pipelines now collect `pending_writes_v1`/`pending_writes_v2` during `asyncio.gather`, then batch-execute `update_article_llm()` sequentially after the gather completes. This eliminates "database is locked" SQLite contention under concurrent article load.

### Fix #5: Event loop leak eliminated
- `_translate_title_sync` now uses `asyncio.get_event_loop()` instead of `asyncio.new_event_loop()` per call
- Added `_translate_titles_batch_async()` to pre-translate all titles before template rendering
- `render_report_v2` and `render_report` pre-populate `_title_translate_cache` via batch async before calling `template.render()`

### Fix #4: Batched line translation
`_translate_report_async` now batches 10 lines per LLM call instead of O(n) calls. A 200-line report now uses ~20 LLM calls instead of 200. Link lines (containing `]((`, `](http`, `](https`) are preserved without translation. Graceful fallback to original lines on LLM error.

### Pre-existing bug fixed
Line 237: `centroids[other]` where `other` is a cluster list → `centroids[j]` (correct integer index). This unblocked report generation.
